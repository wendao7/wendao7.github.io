网络-01-TCP/IP网络模型
win Lv8
2021-05-28 19:32:20
  • 网络
  • TCP/IP模型

1 应用层

专注于为用户提供应用功能，不用关心数据是如何传输的
工作在用户态，传输层及以下工作在内核态

2 传输层

为应用层提供网络支持，让其作为应用间数据传输的媒介，帮助实现应用到应用的通信；

  • 当传输层的数据包大小超过MSS(TCP最大报文段长度)，就要将数据包分块。在TCP协议中，每个分块称为一个TCP段（TCP Segment）
  • 通过端口区分不同应用，由于传输层的报文中会携带端口号，因此接收方可以识别出该报文是发送给哪个应用的

2.1 TCP

传输层控制协议(Transmission Control Protocol)，提供可靠传输

2.2 UDP

不可靠传输，实时性相对较好，传输效率高

3 网络层

实际数据传输功能，负责将数据从一个设备传输到另一个设备，用IP地址对设备进行编号

3.1 IP协议

将传输层的报文作为数据部分，再加上IP包头部组成IP报文。如果IP报文大小超过MTU(以太网一般为1500字节)则再次进行分片，得到一个即将发送到网络的IP报文

3.2 寻址

  • 网络号：标志该IP地址属于哪个子网
  • 主机号：标志同一子网下的不同主机

3.3 路由

实际场景中，两台设备是通过很多网关、路由器、交换机等众多网络设备连接起来的，会形成很多网络的路径，当数据包到底一个网络节点，需要决定下一步走哪条路径
IP 协议的寻址作用是告诉我们去往下一个目的地该朝哪个方向走，路由则是根据「下一个目的地」选择路径

4 数据链路层

为网络层提供链路级别传输的服务：标志网络中的设备，让数据在一个链路中传输

  • MAC
    • 每一台设备的网卡都有一个MAC地址，用来唯一标志设备
    • 路由器计算下一个目的地IP地址，通过ARP协议找到该目的地的MAC地址

5 物理层

为数据链路层提供二进制传输的服务

  • 把数据包转换成电信号，让其可以在物理介质中传输
